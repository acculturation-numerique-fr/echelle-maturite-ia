## 1. Objectif du document

- la méthodologie ;

- les réponses aux clients ;

## 5. Enseignements utiles pour le diagnostic

### Enseignement 1 — Le score ne suffit pas

Les référentiels sérieux ne se limitent pas à une note globale.  
Ils structurent les compétences en domaines.

Application :

- conserver le score /20 ;
- ajouter les dimensions ;
- proposer des axes de progression.

---

### Enseignement 2 — La littératie IA combine usage et recul critique

Un bon utilisateur IA n'est pas seulement quelqu'un qui utilise des outils.  
Il doit aussi comprendre les limites, les hallucinations, la sécurité et la conformité.

Application :

- maintenir les critères liés à la vérification ;
- maintenir RGPD / AI Act ;
- intégrer des messages de vigilance dans les recommandations.

---

### Enseignement 3 — L'adoption est progressive

Les utilisateurs passent généralement par des paliers :

```txt
découverte → expérimentation → usage régulier → structuration → automatisation → pilotage
```

Application :

- maintenir les niveaux Novice à Expert ;
- associer une prochaine étape claire à chaque niveau.

---

### Enseignement 4 — Le diagnostic peut devenir un outil stratégique

Une fois anonymisées et agrégées, les réponses peuvent produire une vision utile :

- niveau moyen par métier ;
- niveau moyen par taille d'entreprise ;
- dimensions les plus faibles ;
- besoins de formation récurrents.

Application V2 :

- benchmark ;
- rapport périodique ;
- contenu LinkedIn ;
- offres de formation ciblées.

## 7. Sources à ne pas surutiliser

Éviter de transformer l'outil en document académique.

Le diagnostic doit rester :

- simple ;
- praticable ;
- orienté action ;
- lisible par des non-spécialistes.

Les références servent la crédibilité, pas l'interface principale.
